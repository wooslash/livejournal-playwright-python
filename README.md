# LiveJournal E2E — Python · Playwright · pytest

End-to-end suite that logs in to **livejournal.com** and publishes a new entry.
Page Object Model, cross-browser via the `--browser` flag, a Faker data factory,
per-step assertions, RU/EN localization, and Allure reporting.

**Repositories:** [Python · Playwright](https://github.com/wooslash/livejournal-playwright-python) · [C# · Selenium](https://github.com/wooslash/livejournal-selenium-csharp)

- **Title** = the author's name — `Dmitriy Shin` (override with `POST_TITLE`).
- **Body** = generated by Faker (unique per run).

## Requirements

- Python 3.12+
- A LiveJournal account **with a confirmed email**. `LJ_USERNAME` is the
  **username**, not the email.
- Optional: [Allure CLI](https://allurereport.org/docs/install/) to render reports.

## Setup

```bash
uv venv && source .venv/bin/activate
uv pip install pytest playwright pytest-playwright faker allure-pytest python-dotenv
python -m playwright install          # download browser engines
cp .env.example .env                  # set LJ_USERNAME (username, not email) + LJ_PASSWORD
```

## Run

```bash
pytest                                # chromium, defaults from .env
pytest --browser firefox              # cross-browser
pytest --browser webkit
pytest --headed                       # visible window
```

Filter into suites with markers:

```bash
pytest -m smoke
pytest -m e2e
pytest -k publish
```

## Configuration

Environment / `.env` (browser is the `--browser` flag, not here):

| Variable          | Values         | Default        |
|-------------------|----------------|----------------|
| `LJ_USERNAME`     | required       | —              |
| `LJ_PASSWORD`     | required       | —              |
| `POST_TITLE`      | any string     | Dmitriy Shin   |
| `LANGUAGE`        | En, Ru         | En             |
| `HEADLESS`        | true, false    | false          |
| `TIMEOUT_SECONDS` | seconds        | 30             |

## Reporting (Allure)

`allure serve` renders the results and **opens the report in your browser
automatically**:

```bash
pytest --alluredir=allure-results
allure serve allure-results
```

Or run the tests and open the report in one step:

```bash
./run.sh                      # pytest, then allure serve (auto-opens the browser)
./run.sh --browser firefox    # extra args pass through to pytest
```

Each test step is an Allure step; a screenshot is attached automatically on
failure. Needs the [Allure CLI](https://allurereport.org/docs/install/)
(`brew install allure`).

## Layout

```
pyproject.toml            deps + pytest config (markers, pythonpath)
conftest.py               fixtures: config, locale, headless, timeout, screenshot-on-fail
src/
  config.py               env/.env config
  localization.py         locale + localized publish-button labels (RU verified, EN best-effort)
  data_factory.py         Faker-generated PostData
pages/
  base_page.py            page plumbing + consent dismissal
  login_page.py           /login.bml — id/class locators, fail-fast on bad username
  post_editor_page.py     /post/ editor — textarea title, Draft.js body, 2-step publish
tests/
  test_publish_entry.py   the E2E test, @smoke/@e2e markers, per-step assertions
```

## Design notes

- **Playwright auto-waits** on every action; the only knob is the per-action
  timeout (`page.set_default_timeout`), set once from config. No implicit/global waits.
- **Localization scope is tiny** — login (element ids), title (only `<textarea>`),
  body (Draft.js class) are language-independent. Only the two publish buttons are
  text-matched, and they match **both** RU and EN labels so a logged-in editor's
  interface language (set by the account, not the flag) doesn't matter.
- **EN publish labels are best-effort** translations of the verified Russian ones
  (`src/localization.py`).
